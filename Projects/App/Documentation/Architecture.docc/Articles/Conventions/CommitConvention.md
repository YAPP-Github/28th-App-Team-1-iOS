# Commit Message 가이드

커밋·PR 메시지 규칙의 **단일 소스는 리포 루트 `CONTRIBUTING.md` §1.2** 다. 팀 협업 프로세스 규칙이라 코드 심볼과 무관 → DocC 가 아니라 거기서 관리한다. (이 문서는 Xcode 에서 길을 잃지 않게 두는 포인터)

## 요약

`type: 설명_부연` — 제목 1줄 한국어. type ∈ `feat` / `fix` / `refactor` / `docs` / `test` / `chore`. 두 type 에 걸치면 커밋을 쪼갠다. 본문은 "무엇이 바뀌었는가"를 bullet 로, 필요할 때만.

상세 규칙·예시·PR 형식 → 리포 루트 `CONTRIBUTING.md`.
